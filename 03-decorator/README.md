# Decorator Pattern

## 특징

1. 객체에 추가적은 요건을 동적으로 더한다. 데코레이터는 서브클래스를 만드는 것을 통해서 기능을 유연하게 확장한다.

2. 데코레이터 패턴은 구상 구성요소를 감싸는 데코레이터들을 사용한다.

3. 데코레이터 클래스 형식은 그 클래스가 감싸고 있는 클래스의 형식을 반영한다.

4. 구성요소를 감싸는 데코레이터의 개수 제한은 없다

5. 데코레이터를 너무 많이 사용하면 코드가 복잡해진다.
